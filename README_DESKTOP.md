# 惠康堂中医处方系统 - 桌面版

## 项目概述

本项目将现有的网页版中医处方系统改造为 Windows 桌面应用，解决了浏览器权限限制问题，实现了处方图片的自动本地归档。

## 核心功能

### 📁 自动图片本地存储（核心需求）
- **自动创建根目录**：首次安装启动时，在系统文档目录自动创建 `惠康堂处方图片` 文件夹
- **按月自动归档**：处方图片自动保存到 YYYY-MM 格式的月度子文件夹（例如：`2026-06`）
- **静默保存**：无弹窗、无需手动选择保存路径，自动保存
- **云端只存文字**：云端仅保存处方文字数据，所有图片仅存储在本地

### 📱 双端运行
- **电脑端**：使用 Electron 桌面应用，享受本地文件系统权限
- **手机端**：继续使用原版网页应用，保持不变

### 🛡️ 全功能保留
- ✅ 多用户权限管理（管理员、普通操作员）
- ✅ 分布式处方编号自动生成
- ✅ 历史处方检索与修改
- ✅ 云端数据同步（KV 存储）
- ✅ 原有全部业务逻辑保持不变

## 开发指南

### 前置要求
- Node.js 16+ 环境
- npm 或 yarn 包管理器

### 本地开发运行

1. 安装依赖
```bash
npm install
```

2. 启动开发模式
```bash
npm start
```

### 打包为 Windows EXE

1. 打包为安装程序（带安装向导）
```bash
npm run build:win
```

2. 打包为便携版（无需安装，直接运行）
```bash
npm run build:win:portable
```

打包完成的文件会输出到 `dist` 目录。

## 技术实现

### Electron IPC 通信

桌面版通过预加载脚本（`preload.js`）暴露安全的 API：

```javascript
// 在网页代码中调用
if (window.electronAPI && window.electronAPI.isElectron) {
    // 桌面版：使用本地文件系统
    const result = await window.electronAPI.savePrescriptionImage(imageData, fileName);
} else {
    // 浏览器版：使用传统下载
    downloadViaBrowser(canvas, fileName);
}
```

### 主要 IPC 接口

| 接口 | 说明 |
|------|------|
| `save-prescription-image` | 保存处方图片到本地 |
| `get-image-directory` | 获取图片保存目录路径 |
| `open-image-directory` | 打开图片保存目录（文件管理器） |

## 部署注意事项

1. **云函数保持不变**：原有的 Cloudflare Functions 后端完全不变
2. **用户数据同步**：多端登录时，新设备可从云端拉取文字处方并重新生成本地图片
3. **权限隔离**：不同用户的处方数据保持隔离，互不干扰

## 常见问题

### Q: 如何更换图片保存目录？
A: 目前版本使用固定路径 `[用户文档]/惠康堂处方图片`，未来可在设置中自定义。

### Q: 如何在多台电脑间同步处方图片？
A: 处方图片仅存本地，您可以手动同步整个 `惠康堂处方图片` 文件夹。

### Q: 手机端和电脑端数据是否同步？
A: 文字处方数据通过云端同步，图片各自保存在本地。
