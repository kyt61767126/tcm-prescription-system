# 云端版开发规范

> 本文档记录云端版（tcm-prescription-system）药品管理模块的永久性开发规范，所有后续修改须遵循以下标准。

## 一、Excel/CSV 导入解析规范

### 1.1 列对应关系（严格）

| 列 | 字段 | 说明 |
|----|------|------|
| A 列 (row[0]) | name | 药品名称 |
| B 列 (row[1]) | code | 简码 |
| C 列 (row[2]) | costPrice | **进价**（用户称 purchasePrice） |
| D 列 (row[3]) | price | 单价 |
| E 列 (row[4]) | unit | 单位 |
| F 列 (row[5]) | dosage | 常用用量 |
| G 列 (row[6]) | stock | 库存 |

### 1.2 进价校验规范

- C 列为空或非数字时：**禁止自动填充 0**，将 `costPrice` 设为 `null`
- 记录异常药材名称到 `missingPriceMedicines` 列表
- 统一使用 `parseMedicineRow(row)` 辅助函数解析，确保 Excel/CSV/JSON 三种导入路径行为一致

### 1.3 导入完成统计弹窗规范

导入完成后必须展示统计信息：
- 成功导入条数（进价完整）
- 进价缺失条数
- 异常药材名称列表（如有）

### 1.4 导入后自动同步

- 导入完成后调用 `saveMedicinesToCloud(medicines)` 保存到云端
- 保存成功后调用 `getMedicinesFromCloud(true)` 强制刷新（绕过缓存）
- 重新渲染药品列表，无需手动刷新页面

## 二、前端渲染规范

### 2.1 进价字段渲染

- **禁止固定写死进价 0 默认值**
- `costPrice` 为 `null`/`undefined`/空字符串时显示红色"未设置"提示
- 有值时正常显示数值 + "元"

### 2.2 编辑/新增药品进价输入

- 编辑药品时，进价 prompt 默认值使用现有值（无值时为空字符串，非 '0'）
- 新增药品时，进价 prompt 无默认值
- 用户留空时 `costPrice` 设为 `null`，非 `0`

## 三、缓存优化规范

### 3.1 URL 时间戳

- `getMedicinesFromCloud` 和 `getFormulasFromCloud` 的请求 URL 必须拼接 `?_t=${Date.now()}` 时间戳
- 目的：绕过浏览器缓存，解决手机端价格不更新问题

### 3.2 刷新药品库按钮

- 药品管理弹窗必须包含【刷新药品库】按钮（绿色）
- 点击后调用 `refreshMedicineLibrary()` 函数
- 该函数强制从云端拉取最新数据（`forceRefresh=true`）并重新渲染

### 3.3 数据变更后云端同步

以下操作完成后必须调用 `saveMedicinesToCloud(medicines)`：
- 新增药品（`showAddMedicineForm`）
- 编辑药品（`editMedicine`）
- 删除药品（`deleteMedicine`）
- 清空药物库（`clearMedicineLibrary`）
- 批量出入库（`showBatchStockForm`，仅成功时）
- 导入药品（`processImportedMedicines`）

## 四、验收核对项

- [x] Excel C 列进价数值可完整解析存入数据库，不再默认变成 0
- [x] 药品管理列表正常展示对应进价，和 Excel 表格数据一致
- [x] 导入出现进价空白、格式错误弹出提示，不会静默填充 0
- [x] 导入完成自动刷新页面，手机端打开直接看到更新后的进价、单价
- [x] 页面新增一键刷新药材按钮，规避手机浏览器缓存延迟

## 六、关键函数索引

| 函数 | 职责 |
|------|------|
| `parseMedicineRow(row)` | 解析单行数据，严格校验 C 列进价 |
| `importMedicines(event)` | Excel/CSV/JSON 导入入口 |
| `processImportedMedicines(imported, missing)` | 处理导入动作 + 统计弹窗 + 云端同步 |
| `renderMedicineList()` | 渲染药品列表（进价缺失显示"未设置"） |
| `getMedicinesFromCloud(force)` | 从云端获取药品（URL 带时间戳） |
| `saveMedicinesToCloud(medicines)` | 保存药品到云端 |
| `refreshMedicineLibrary()` | 一键刷新药品库 |
